# K-Ride 온보딩 UX 원본 기획 명세

> 원본 출처: `.ai/kride.txt` — 초기 기획 요구사항 (사용자 작성)

---

## 기존 화면 (유지)

- `/login` — 로그인
- `/signup` — 회원가입

---

## INTRO1 — 여행 기간 선택 (`/browse`)

- 검은 배경
- 당일치기 / 1박 2일 / 2박 3일 버튼 3개
- 버튼 클릭 → `kride_form.duration` 저장 + 다음 화면 이동

---

## INTRO2 — 아티스트 선택 (`/movies`)

- 좋아하는 가수/아이돌/배우 사진 카드 표시
- 선택 조건:
  - 1개 이상 선택해야 다음 진행 가능
  - 5개 이하 제한 (5개 초과 클릭 시 추가 불가)
  - 1개 미만이면 "1개 이상 골라달라" 안내
- circle 버튼 → 화살표 버튼(KRIDE_NEXT_BTN)으로 다음 화면 이동
- `kride_form.selectedArtists` 저장

---

## INTRO3 — 지역 선택 (`/latest`)

- 서울, 경기, 경북, 경남, 전북, ..., 제주 등 대표 지역 사진 카드
- 선택 조건:
  - 1개 이상 선택해야 다음 진행 가능
  - **2개 이하** 제한 (2개 초과 클릭 시 추가 불가) — *현재 구현: 2개 초과 경고 토스트*
  - 1개 미만이면 "1개 이상 골라달라" 안내
- 화살표 버튼(KRIDE_NEXT_BTN)으로 다음 화면 이동
- `kride_form.selectedRegions` 저장

> **기획 vs 구현 차이**: 원본 기획은 5개 이하였으나, 실제 구현은 지역 2개 이하로 제한함

---

## INTRO4 — 여행 목적 선택

- 검은 배경에 "여행의 목적은?" 제목
- checkbox Select 형태로 목적 선택
- **단일 선택** (1개만 선택 가능)
- `kride_form.purposes` 저장

---

## INTRO5 — 예산 설정

- "내가 생각하는 여행 예산은?" 제목
- 최소 3만원 ~ 최대 200만원 범위
- 빨간색 DualRangeSlider
- "AI 여행 추천 받기" 버튼 → MY_LIST로 이동
- `kride_form.budget` 저장

---

## MY_LIST 페이지 (`/my-list`)

INTRO1~5에서 선택한 내용 요약 표시:
- 선택한 기간, 아티스트, 지역, 목적, 예산

AI 추천 배너:
```
AI기반으로 방문 경로를 추천해드려요.
전주 한옥마을을 추천드립니다
```

배너 클릭 → FOCUS 페이지로 이동

---

## FOCUS 페이지 (`/focus`)

레이아웃: **왼쪽 60% 지도 + 오른쪽 40% 사이드 패널**

- 왼쪽: 여행지 지역 포커스 지도 + 추천 경로(노드) + 주변 맛집 마커
- 오른쪽: 추천 여행 경로 일정 패키지 (날짜별 아침/오후 구성)

---

## 화면 흐름 요약

```
/login or /signup
        ↓
INTRO1 (/browse) — 여행 기간 선택
        ↓
INTRO2 (/movies) — 아티스트 선택
        ↓
INTRO3 (/latest) — 지역 선택
        ↓
INTRO4           — 목적 선택
        ↓
INTRO5           — 예산 설정
        ↓
MY_LIST (/my-list) — 선택 요약 + AI 배너
        ↓
FOCUS (/focus)   — 지도 + 추천 일정
```
